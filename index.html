Inventory Tracker

This full-stack Android application was created for a mobile programming class in late 2025. Its purpose is to track items in an inventory in order to hypothetically bring administrative value to an organization that needs to manage inventories. The application was a strong choice for enhancement due to its lack of cloud connectivity. In such an app, cloud services are highly necessary in order to deliver value.

This software demonstrates thoughtful UI design, sound software engineering and design principles, REST API design and implementation, and usage of robust database technology that powers massive, real-world enterprise software.

Enhancing this project was a great exercise in full-stack development. Lessons were learned across Snowflake, AWS, and Android ecosystems. Software engineering and design best practices were followed throughout and served as a practical exercise for producing well-architected code that can be extended later. General system knowledge of Snowflake and AWS has been improved significantly. The greatest challenge posed was creating an entire backend API in approximately two weeks.

API for Snowflake

The REST API backend, hosted with AWS Lambda, provides a robust platform for transferring data to and from the Snowflake database securely. The API sends and receives data primarily using the HTTP body to facilitate safe transport of sensitive data. In addition, the backend hashes passwords using SHA-256 and the database only stores the hashed passwords to protect users in the event of a data breach and to ensure that even administrators cannot view users' passwords.

https://github.com/collinogren/Inventory-Tracker-Snowflake-API

Enhancements Made

  • Added a flexible and remote Snowflake database so that users can access their accounts from any device
  • Added an AWS Lambda backend to facilitate secure communication
  • Added unit tests to build confidence in existing systems
  • Improved searching algorithm by using the Jaro Winkler similarity algorithm provided by Snowflake
  • Utilize a service structure to separate the concerns of communicating with the API from the controllers
  • Moved add and subtract item quantity buttons into the edit activity due to the new concerns of latency

Auto Team Totals

Created in 2023 for use at U.S. Figure Skating competitions, this software provides value in immense time savings for calculating team points at U.S. Figure Skating competitions which previously had to be done manually. It functions by parsing HTML files and placing the results into specific data structures for various algorithms to manipulate and retrieve the results for the competition. This project involved significant consultation with stakeholders in U.S. Figure Skating to fully understand and iron out the requirements.

The project demonstrates the implementation of robust and effective data structures and algorithms that solve real-world problems. The use of the Rust programming language demonstrates a commitment to software speed, safety, and security.

This project served as a reminder to always assess algorithmic efficiency and to continue to look for ways to optimize and simplify algorithms. Minor challenges included fighting the Rust borrow checker on specific changes that required moving code into separate functions. These were short-lived issues, however.
