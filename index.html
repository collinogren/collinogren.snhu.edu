Inventory Tracker

This full-stack Android application was created for a mobile programming class and demonstrates thoughtful UI design, sound software engineering and design principles, REST API design and implementation, and usage of robust database technology that powers massive, real-world enterprise software.

Android Frontend App

The frontend Android app showcases software engineering and design principles such as separation of concerns with its model-view-controller architecture. It also demonstrates unit testing, multithreading, and connectivity to a REST API backend.

https://github.com/collinogren/CS-360-Inventory-Tracker

API for Snowflake

The REST API backend, hosted with AWS Lambda, provides a robust platform for transferring data to and from the Snowflake database securely. The API sends and receives data primarily using the HTTP body to facilitate safe transport of sensitive data. In addition, the backend hashes passwords using SHA-256 and the database only stores the hashed passwords to protect users in the event of a data breach and to ensure that even administrators cannot view users' passwords.

https://github.com/collinogren/Inventory-Tracker-Snowflake-API

Enhancements Made

  • Added a flexible and remote Snowflake database so that users can access their accounts from any device
  • Added an AWS Lambda backend to facilitate secure communication
  • Added unit tests to build confidence in existing systems
  • Improve searching algorithm by using the Jaro Winkler similarity algorithm provided by Snowflake
  • Utilize a service structure to separate the concerns of communicating with the API from the controllers
  • Move add and subtract item quantity buttons into the edit activity due to the new concerns of latency

Future Considerations

  • If the service goes public, analyze Snowflake usage to optimize warehouse size and session length
